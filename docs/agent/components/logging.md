# ロギング

## 概要

内部的にwasmハイパーバイザのログを記録する。ログはロギングサブシステムで処理される。このログは、最終的にARM/ITMやUARTで外部にテキストとして出力される。

## ロギングAPI

- 辞書参照IPCを用いてロギングを行う。
  - 辞書参照IPCでは辞書はあらかじめ定義されている必要がある。
    - 辞書はconstexprとしてヘッダファイルに定義されている。
  - システム設計時に定義済みのテキストに値を埋め込んだものしか出力することはできない。
  - タイムスタンプはロギングサブシステム側

## ログレベル

ログレベルには下記がある。ログレベルはそれぞれ出力を有効、無効化できる。

|レベル|説明|
|---|---|
|TRACE|トレースログ|
|DEBUG|デバッグログ|
|INFO|正常な情報ログ|
|ERROR|停止が不要な問題のグ|
|FATAL|停止が必要な問題のログ|
